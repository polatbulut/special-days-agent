special-days-agent — architecture schemas

A CLI pipeline that collects forward-looking special dates (holidays, ticketed events, football fixtures, trade fairs) across Turkey and destination markets, normalizes them into one canonical record, and enriches each with a nearest airport, a 0–100 THY ticket-sales impact score, a köprü bridge range, and per-day demand curves.

The four schemas below are the visual overview. For the exact formulas, constants, edge cases and worked examples behind every step, see the calculation reference.

1 · Pipeline backbone

Every run is the same four-stage spine — collect, filter, enrich, output — and one canonical SpecialDate record rides through all of it.

2 · Collection topology

Both agents call the same six sources. TurkeyAgent (solid) reaches all six; InternationalAgent (dashed) only the shared four. Each source hits a live API (coral) or a bundled dataset (gray).

3 · Enrichment chain

Collected rows are de-duplicated, the noise filter trims long spans, then enrich() runs four fixed-order steps. Airport mapping comes first because both scorers read the distance.

4 · Pluggable impact scorer

Behind step 2: get_scorer returns the offline heuristic or an LLMScorer fed by one OpenAI-compatible gateway fronting three backends.
